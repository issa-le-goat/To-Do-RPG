Titre du Projet

Conception, Choix Technologiques et Sécurité

Vision Globale

Ce projet consiste à développer une application web robuste et intuitive, en mettant l'accent sur les performances techniques et la sécurité des données.

L'objectif principal est de fournir une solution fiable tout en gardant une maîtrise complète sur l'architecture et le code source.

Objectifs Techniques

  • Simplicité et Performance : Un code léger permettant un chargement instantané.
  • Maintenabilité : Une architecture claire et sans dépendances superflues.
  • Sécurité : Une protection rigoureuse des données sensibles des utilisateurs.

Stack Frontend : Le choix du "Vanilla"

L'interface a été entièrement développée avec les technologies standards du web (HTML, CSS, JS), sans l'utilisation de frameworks externes.

  • HTML : Structure sémantique et accessibilité.
  • CSS : Mise en page adaptative et design épuré.
  • JavaScript : Logique interactive et gestion dynamique du DOM.

Pourquoi se passer de Frameworks ?

  • Contrôle total : Compréhension précise de chaque ligne de code exécutée.
  • Légèreté : Absence de bibliothèques lourdes à charger.
  • Pérennité : Le code standard (Vanilla) ne devient jamais obsolète et ne nécessite pas de mises à jour de dépendances complexes.

Architecture et Sécurité

Si le frontend garantit l'expérience utilisateur, le backend a la responsabilité de la sécurité de l'application.

Le point critique de tout système d'authentification est la gestion et le stockage des informations d'identification.

L'enjeu du stockage des mots de passe

Règle d'or en cybersécurité : Il ne faut jamais stocker un mot de passe en clair dans une base de données.

Les mots de passe doivent subir une transformation mathématique irréversible appelée hachage. Ainsi, même en cas de fuite de la base de données, les mots de passe originaux restent illisibles.

Qu'est-ce que Bcrypt ?

Bcrypt est un algorithme de hachage cryptographique spécialement conçu pour les mots de passe. Il intègre deux mécanismes fondamentaux :

  • Le Sel (Salt) : Une chaîne de caractères aléatoire ajoutée au mot de passe avant le hachage. Cela empêche les attaques par dictionnaire (Rainbow Tables).
  • Le Facteur de Coût (Cost) : Un mécanisme qui ralentit volontairement le calcul du hachage.

Pourquoi Bcrypt plutôt que MD5 ou SHA ?

MD5 / SHA-256 Bcrypt
Conçus pour être ultra-rapides (vérification de fichiers). Conçu pour être lent par nature.
Un attaquant peut tester des milliards de combinaisons par seconde (Brute Force). Le calcul lent empêche physiquement le Brute Force à grande échelle.
Vitesse fixe. Le facteur de coût est ajustable pour s'adapter à l'évolution de la puissance matérielle.
